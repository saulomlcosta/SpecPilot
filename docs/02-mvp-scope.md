# 02 - Escopo do MVP

## O que entra

- cadastro de usuario
- login simples
- criacao de projeto
- registro da descricao inicial da ideia
- geracao de perguntas de refinamento
- resposta das perguntas pelo usuario
- geracao do documento inicial

## Documento inicial previsto

O documento final do MVP deve conter:

- visao geral
- requisitos funcionais
- requisitos nao funcionais
- casos de uso
- riscos

## O que nao entra

- RAG
- upload de arquivos
- PDF
- chat livre
- microservicos
- mensageria
- multiplos agentes
- integracao com GitHub
- geracao automatica de codigo
- dashboards complexos
- colaboracao entre varios usuarios no mesmo projeto

## Regra de controle de escopo

Toda decisao futura deve responder a pergunta:

> Isto e necessario para o usuario cadastrar um projeto, refinar a ideia com IA e obter a documentacao inicial?

Se a resposta for nao, a funcionalidade deve ficar fora da etapa atual.
