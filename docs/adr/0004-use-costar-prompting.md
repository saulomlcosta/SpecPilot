# ADR 0004 - Usar prompting CO-STAR

## Status

Aceita

## Contexto

Os prompts precisam ser claros, auditaveis e reproduziveis.

## Decisao

Padronizar prompts de runtime com o metodo CO-STAR.

## Alternativas consideradas

- prompts ad hoc sem estrutura
- outros frameworks de prompt engineering

## Justificativa

CO-STAR oferece uma estrutura simples e facil de explicar em contexto academico, sem aumentar demais o custo de manutencao dos prompts.

## Consequencias

- maior consistencia
- melhor explicabilidade
- mais facilidade de manutencao
