# 01 - Declaracao do problema

## Problema central

Na fase inicial de ideacao de software, e comum que a descricao do problema seja vaga. Isso gera dificuldades para transformar uma ideia em requisitos claros, casos de uso coerentes e riscos identificaveis.

## Consequencias praticas

- retrabalho por entendimento incompleto
- requisitos contraditorios
- inicio prematuro da implementacao
- documentacao pobre ou inexistente

## Hipotese do projeto

Se a IA Generativa for usada para conduzir o refinamento da ideia e estruturar uma primeira documentacao, entao o usuario pode obter uma base mais consistente para discutir, validar e iniciar o desenvolvimento.
